# RickyData Docs — Project Context

## Stack
- **Docs site**: Docusaurus 3.9 at `apps/docs-site/` (TypeScript, MDX, Privy wallet auth)
- **Legacy SPA**: `apps/docs-web/` (Vite + React Router — being replaced)
- **Backend**: Express API at `apps/docs-api/`
- **Packages**: `packages/content-parsers`, `packages/shared-types`
- **SDK integration**: `rickydata`, `@rickydata/react`, `@rickydata/chat` (interactive components embedded in MDX)

## Build Command
```bash
cd apps/docs-site && npx docusaurus build
```

## Design System

- **Typography**: Space Grotesk (headings, 600-700 weight) + DM Sans (body, 400-500 weight) + IBM Plex Mono (code). NO Inter.
- **Color**: OKLCH palette mapped to Docusaurus Infima variables (`--ifm-*`) and RickyData tokens (`--rd-*`). Warm amber/ochre accent at hue 70. NO emerald/teal.
- **Dark theme**: Default. Warm chroma backgrounds (hue ~60, chroma 0.005-0.008), never pure gray.
- **No Tailwind CSS**: Pure CSS custom properties only. Tailwind breaks Docusaurus layout (learned from KFDB).
- **SDK components**: Use inline `CSSProperties` with `--rd-*` tokens. Wrap in `BrowserOnly` for SSG safety.

## Webpack SDK Resolution

The `docusaurus.config.ts` includes a critical webpack plugin (`resolve-rickydata-subpaths`) that:
1. Aliases `rickydata/agent`, `rickydata/wallet`, etc. to their dist files (the npm package only exports `import` conditions, not `require`)
2. Rewrites `node:` protocol URIs to bare module names (the SDK uses `node:fs`, `node:os`, etc. which webpack 5 can't handle)

Do NOT remove this plugin or the build will fail.
